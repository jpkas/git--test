Html-Css-Javascript Web Page Portofolio.

Power of div,id and class(es).

By JeanPierre Kasonga

This is an html-css-javascript web page that I designed just to show what I can do with this technology. I did not use bootstrap.Don't try to find on it the Mobile-First features, even though it is responsive at some extent. Being myself beginner web developer, I would like to encourage those who just began to learn html-CSS-Javascript . At the same time I would like to thank nucamp Bootcamp for the opportunity to code offered to me.
The flow of my html work is based on a judicious choice of div(s). A first div contains the h1,h2,h3 elements. A secomd div contains the paragraph of introduction .Then I have created a section containing 4 div(s) to hold two lists and two images. Outside of the section two other div(s) hold a table and a fieldset with form.
Without CSS the html page is vertical each element mentioned above following the other without colors nor movement. I like to compare html structure to the human skeleton without flesh. In art,the skeleton evokes the idea of ghost or dead person. It is scary. I mean that html alone is boring. But a human body without skeleton cannot stand. To embelish my html-page, I have created a css-file (index.css) in the same folder where I placed the html-file(index.html) and the two images files. I have chosen class(es) and Id(s) selectors to apply property on the html elements. The class center ,for example ,enabled me to change the html vertical layout into an horintal layout by the mean of the display-flex property. In the same way I applied color and background-color to diffrent parts of the page. Now the page looks like pretty.
However it is lacking movement or interactivity. That is why I invited Javascript to help. Instead of creating a Javascript file, I embeded my script in the html placing it before the body's closing tag. The Javascript code inside the script has the power to bring motion to the html-css-page. Of course ,I linked the 4 buttons to the id(s) of each element to be toggled. With the power of javascript addEventListener code the computer is not only able to listen to the click made upon the buttons, but to obey to that prompt. In obedience to the prompt , the computer will make the specified element appear or disappear from the page. So ,by clicking on the buttons one after another you will remove from the page the lists , images and table ; only to leave on it the buttons, the introducting and concluding texts and the form. Vis-versa, you can bring back these elements again just by clicking the same buttons one after another.It is almost magic, isn't ?
